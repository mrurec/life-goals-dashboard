Зачем этот модуль: интеграция всех технологий Why This Module: Integration of All Technologies

You've completed 16 modules: Kotlin, Spring Boot, GraphQL, React, Relay. Now build a REAL widget that integrates everything. Let's use a simple example — Reading Tracker (book reading progress tracking) — and walk through the entire path from database schema to interactive React component.

Reading Tracker: пользователь добавляет книги, логирует reading sessions (сколько страниц прочитал), видит прогресс (%), стоимость его времени. Это меньше, чем Fitness Tracker, но содержит ВСЕ паттерны Meta Engineering: Entity ↔ GraphQL ↔ Relay, N+1 prevention через DataLoaders, mutation error handling, cursor pagination, cache invalidation, ViewerContext on every layer. Reading Tracker: users add books, log reading sessions (pages read), view progress (%), and see reading streaks. It's simpler than the Fitness Tracker but contains ALL Meta Engineering patterns: Entity ↔ GraphQL ↔ Relay, N+1 prevention via DataLoaders, mutation error handling, cursor pagination, cache invalidation, ViewerContext on every layer.

    cover_url TEXT,
    rating INT CHECK (rating >= 0 AND rating <= 5),
    started_at DATE,
    finished_at DATE,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),

);

CREATE INDEX idx_book_user_id ON book(user_id);
CREATE INDEX idx_book_status ON book(user_id, status);
CREATE INDEX idx_book_created_at ON book(user_id, created_at DESC);

CREATE TABLE reading_session (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    book_id UUID NOT NULL REFERENCES book(id) ON DELETE CASCADE,
    user_id UUID NOT NULL REFERENCES "user"(id) ON DELETE CASCADE,
    pages_read INT NOT NULL CHECK (pages_read > 0),
    duration_minutes INT,
    session_date DATE NOT NULL,
    notes TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_reading_session_book_id ON reading_session(book_id);
CREATE INDEX idx_reading_session_user_id ON reading_session(user_id, session_date DESC);
ВАЖНО: индексы на (user_id, ...) потому что ВСЕГДА фильтруем по пользователю. Это ключ к быстрым запросам в multi-user системе. IMPORTANT: indexes on (user_id, ...) because we ALWAYS filter by user. This is the key to fast queries in multi-user systems.

package com.mrurec.lifegoals.reading.model

import jakarta.persistence.*
import java.time.LocalDate
import java.time.Instant
import java.util.UUID

enum class BookStatus {
    READING,
    COMPLETED,
    DROPPED,
    WANT_TO_READ
}

@Entity
@Table(name = "book", indexes = [
    Index(columnList = "user_id"),
    Index(columnList = "user_id,status"),
    Index(columnList = "user_id,created_at DESC")
])
data class Book(
    @Id
    val id: UUID = UUID.randomUUID(),

    val userId: UUID,
    val title: String,
    val author: String? = null,
    val totalPages: Int,

    @Column(name = "current_page")
    var currentPage: Int = 0,

    @Enumerated(EnumType.STRING)
    var status: BookStatus = BookStatus.WANT_TO_READ,

    val coverUrl: String? = null,
    var rating: Int? = null,

    var startedAt: LocalDate? = null,
    var finishedAt: LocalDate? = null,

    val createdAt: Instant = Instant.now(),

    @Version
    var updatedAt: Instant = Instant.now()
) {
    // Вычисляемое поле: прогресс как %
    val progressPercent: Int
        get() = if (totalPages == 0) 0 else (currentPage * 100) / totalPages
}

@Entity
@Table(name = "reading_session")
data class ReadingSession(
    @Id
    val id: UUID = UUID.randomUUID(),

    val bookId: UUID,
    val userId: UUID,
    val pagesRead: Int,
    val durationMinutes: Int? = null,
    val sessionDate: LocalDate,
    val notes: String? = null,

    val createdAt: Instant = Instant.now()
)

@DgsComponent
class BookFieldResolver {
    @DgsData(parentType = "Book", field = "readingSessions")
    fun readingSessions(dfe: DgsDataFetchingEnvironment): CompletableFuture<Connection<ReadingSession>> {
        val book = dfe.getSource<Book>()
        val first = dfe.getArgument<Int>("first") ?: 10
        val after = dfe.getArgument<String>("after")

        val loader = dfe.getDataLoader<UUID, List<ReadingSession>>("readingSessionsByBookLoader")
        return loader.load(book.id).thenApply { sessions ->
            ConnectionBuilder.build(sessions, first, after)
        }
    }

    @DgsData(parentType = "Book", field = "progressPercent")
    fun progressPercent(dfe: DgsDataFetchingEnvironment): Int {
        val book = dfe.getSource<Book>()
        return book.progressPercent
    }
}

// com/mrurec/lifegoals/reading/dataloader/ReadingDataLoaders.kt
package com.mrurec.lifegoals.reading.dataloader

import com.mrurec.lifegoals.reading.model.ReadingSession
import com.mrurec.lifegoals.reading.repository.ReadingSessionRepository
import com.netflix.graphql.dgs.DgsComponent
import com.netflix.graphql.dgs.DgsDataLoader
import org.dataloader.MappedBatchLoader
import java.util.UUID
import java.util.concurrent.CompletableFuture
import java.util.concurrent.CompletionStage

@DgsComponent
class ReadingDataLoaders {
    @DgsDataLoader(name = "readingSessionsByBookLoader")
    class ReadingSessionsByBookLoader(
        private val readingSessionRepository: ReadingSessionRepository
    ) : MappedBatchLoader<UUID, List<ReadingSession>> {

        override fun load(bookIds: Set<UUID>): CompletionStage<Map<UUID, List<ReadingSession>>> {
            return CompletableFuture.supplyAsync {
                val sessions = readingSessionRepository.findByBookIdIn(bookIds.toList())
                val grouped = sessions.groupBy { it.bookId }
                bookIds.associateWith { bookId ->
                    grouped[bookId] ?: emptyList()
                }
            }
        }
    }
}
"Как ваша система предотвращает N+1 запросы?""How does your system prevent N+1 queries?" — "Мы используем DataLoader паттерн из Meta's TAO. Каждый child field resolver использует DataLoader, который батчит parent IDs. Когда GraphQL разрешает 20 книг с readingSessions, DataLoader собирает все 20 book IDs и выполняет один SQL запрос с WHERE book_id IN (...). Это превращает O(N) запросов в O(1) per level.""We use the DataLoader pattern from Meta's TAO. Every child field resolver uses a DataLoader that batches parent IDs. When GraphQL resolves 20 books with readingSessions, the DataLoader collects all 20 book IDs and issues one SQL query with WHERE book_id IN (...). This turns O(N) queries into O(1) per level."

// com/mrurec/lifegoals/reading/service/BookService.kt
@Service
class BookService(
    private val bookRepository: BookRepository,
    private val readingSessionRepository: ReadingSessionRepository,
    private val cacheService: ReadingCacheService,
    private val notificationService: NotificationService
) {
    @Transactional(readOnly = true)
    fun getBooks(
        viewer: ViewerContext,
        first: Int,
        after: String?,
        filter: BookFilter?
    ): Connection<Book> {
        val cursor = after?.let { CursorCodec.decode(it) }
        val pageable = PageRequest.ofSize(first + 1)

        val books = if (cursor != null) {
            bookRepository.findAfterCursor(
                userId = viewer.userId,
                cursorTime = cursor.createdAt,
                cursorId = cursor.id,
                pageable = pageable
            )
        } else {
            bookRepository.findFirstPage(viewer.userId, pageable)
        }

        val totalCount = bookRepository.countByUserId(viewer.userId)
        return ConnectionBuilder.build(
            items = books,
            first = first,
            totalCount = totalCount,
            cursorExtractor = { book -> CursorCodec.encode(book.createdAt, book.id) }
        )
    }

    @Transactional
    fun addBook(input: AddBookInput, viewer: ViewerContext): AddBookPayload {
        val errors = mutableListOf<UserError>()

        if (input.title.isBlank()) {
            errors.add(UserError("title", "Title cannot be empty", ErrorCode.VALIDATION_ERROR))
        }
        if (input.totalPages <= 0) {
            errors.add(UserError("totalPages", "Total pages must be positive", ErrorCode.VALIDATION_ERROR))
        }
        if (errors.isNotEmpty()) {
            return AddBookPayload(book = null, errors = errors)
        }

        val book = Book(
            userId = viewer.userId,
            title = input.title.trim(),
            author = input.author?.trim(),
            totalPages = input.totalPages,
            status = input.status ?: BookStatus.WANT_TO_READ,
            coverUrl = input.coverUrl,
            startedAt = if (input.status == BookStatus.READING) LocalDate.now() else null
        )

        val saved = bookRepository.save(book)
        cacheService.invalidateWidgetData(viewer.userId)
        return AddBookPayload(book = saved, errors = emptyList())
    }

    @Transactional
    fun logReadingSession(input: LogReadingSessionInput, viewer: ViewerContext): LogReadingSessionPayload {
        val book = bookRepository.findById(input.bookId).orElse(null)

        if (book == null) {
            return LogReadingSessionPayload(
                session = null, book = null,
                errors = listOf(UserError("bookId", "Book not found", ErrorCode.NOT_FOUND))
            )
        }

        if (book.userId != viewer.userId) {
            return LogReadingSessionPayload(
                session = null, book = null,
                errors = listOf(UserError("bookId", "Book not found", ErrorCode.NOT_FOUND))
            )
        }

        if (input.pagesRead <= 0) {
            return LogReadingSessionPayload(
                session = null, book = null,
                errors = listOf(UserError("pagesRead", "Pages must be positive", ErrorCode.VALIDATION_ERROR))
            )
        }

        val session = ReadingSession(
            bookId = book.id,
            userId = viewer.userId,
            pagesRead = input.pagesRead,
            durationMinutes = input.durationMinutes,
            sessionDate = input.sessionDate ?: LocalDate.now(),
            notes = input.notes
        )
        readingSessionRepository.save(session)

        val newPage = minOf(book.currentPage + input.pagesRead, book.totalPages)
        book.currentPage = newPage

        if (newPage >= book.totalPages && book.status == BookStatus.READING) {
            book.status = BookStatus.COMPLETED
            book.finishedAt = LocalDate.now()

            notificationService.publishAchievement(
                userId = viewer.userId,
                widget = "READING",
                message = "Finished reading \"${book.title}\"! 📚"
            )
        }

        bookRepository.save(book)
        cacheService.invalidateWidgetData(viewer.userId)

        return LogReadingSessionPayload(
            session = session,
            book = book,
            errors = emptyList()
        )
    }
}

@Service
class ReadingCacheService(
    private val redisTemplate: RedisTemplate<String, String>,
    private val objectMapper: ObjectMapper
) {
    companion object {
        private const val WIDGET_DATA_TTL = 300L
        private const val STATS_TTL = 600L
    }

    fun getWidgetData(userId: UUID): ReadingWidgetData? {
        val key = "lifegoals:reading:widget_data:$userId"
        return redisTemplate.opsForValue().get(key)?.let {
            objectMapper.readValue(it, ReadingWidgetData::class.java)
        }
    }

    fun setWidgetData(userId: UUID, data: ReadingWidgetData) {
        val key = "lifegoals:reading:widget_data:$userId"
        redisTemplate.opsForValue().set(
            key,
            objectMapper.writeValueAsString(data),
            Duration.ofSeconds(WIDGET_DATA_TTL)
        )
    }

    fun invalidateWidgetData(userId: UUID) {
        val keys = listOf(
            "lifegoals:reading:widget_data:$userId",
            "lifegoals:reading:stats:$userId"
        )
        redisTemplate.delete(keys)
    }
}

  const [commit, isInFlight] = useMutation<LogReadingSessionFormMutation>(mutation);
  const [pagesRead, setPagesRead] = useState('');
  const [fieldErrors, setFieldErrors] = useState<Record<string, string>>({});

  const handleSubmit = useCallback((e: React.FormEvent) => {
    e.preventDefault();
    setFieldErrors({});

    commit({
      variables: {
        input: {
          bookId,
          pagesRead: parseInt(pagesRead, 10),
        },
      },
      onCompleted: (response) => {
        const { errors } = response.logReadingSession;

        if (errors.length > 0) {
          const errorMap: Record<string, string> = {};
          errors.forEach(err => {
            if (err.field) {
              errorMap[err.field] = err.message;
            }
          });
          setFieldErrors(errorMap);
          return;
        }

        onSuccess();
      },
      onError: (error) => {
        console.error('Network error:', error);
        setFieldErrors({ _network: 'Connection error. Please try again.' });
      },
    });
  }, [bookId, pagesRead, commit, onSuccess]);

        <label>Страницы прочитаныPages read</label>

          type="number"
          value={pagesRead}
          onChange={e => setPagesRead(e.target.value)}
          min="1"
          required
        />
        {fieldErrors.pagesRead && (
          <span className={styles.error}>{fieldErrors.pagesRead}</span>
        )}
      </div>
      {fieldErrors._network && (
        <div className={styles.networkError}>{fieldErrors._network}</div>

// com/mrurec/lifegoals/common/auth/ViewerContextProvider.kt
@Component
class ViewerContextProvider {
    fun get(): ViewerContext {
        val auth = SecurityContextHolder.getContext().authentication
            ?: throw AuthenticationException("Not authenticated")

        val jwt = auth.principal as? Jwt
            ?: throw AuthenticationException("Invalid token")

        return ViewerContext(
            userId = UUID.fromString(jwt.getClaimAsString("sub")),
            email = jwt.getClaimAsString("email"),
            permissions = jwt.getClaimAsStringList("permissions")?.toSet() ?: emptySet()
        )
    }
}

// com/mrurec/lifegoals/common/relay/CursorCodec.kt
object CursorCodec {
    fun encode(createdAt: Instant, id: UUID): String {
        val raw = "${createdAt.toEpochMilli()}|$id"
        return Base64.getEncoder().encodeToString(raw.toByteArray())
    }

    fun decode(cursor: String): CursorData {
        val raw = String(Base64.getDecoder().decode(cursor))
        val parts = raw.split("|")
        require(parts.size == 2) { "Invalid cursor format" }
        return CursorData(
            createdAt = Instant.ofEpochMilli(parts[0].toLong()),
            id = UUID.fromString(parts[1])
        )
    }

    data class CursorData(val createdAt: Instant, val id: UUID)
}

object ConnectionBuilder {
    fun <T> build(
        items: List<T>,
        first: Int,
        totalCount: Long,
        cursorExtractor: (T) -> String
    ): Connection<T> {
        val hasNextPage = items.size > first
        val pageItems = items.take(first)

        val edges = pageItems.map { item ->
            Edge(node = item, cursor = cursorExtractor(item))
        }

        return Connection(
            edges = edges,
            pageInfo = PageInfo(
                hasNextPage = hasNextPage,
                hasPreviousPage = false,
                startCursor = edges.firstOrNull()?.cursor,
                endCursor = edges.lastOrNull()?.cursor
            ),
            totalCount = totalCount
        )
    }
}

Паттерны Meta Engineering: почему именно так Meta Engineering Patterns: Why This Way

1. Fragment ColocationFragment Colocation (Relay)

"Почему colocate фрагменты с компонентами?""Why colocate fragments with components?" — "В Meta мы нашли что когда data requirements живут отдельно от компонентов, они расходятся. Кто-то удаляет поле из query но компонент всё ещё его ожидает — crash. Fragment colocation означает что data контракт лежит прямо рядом с кодом, который его использует.""At Meta, we found that when data requirements live separately from components, they drift apart. Someone removes a field from the query but a component still expects it — crash. Fragment colocation means the data contract is right next to the code that uses it."

2. DataLoaders (TAO паттерн)DataLoaders (TAO Pattern)

"Как Meta решает N+1?""How does Meta solve N+1?" — "Meta's TAO система батчит все data fetches в рамках request. DGS DataLoaders реплицируют это: вместо fetch one entity at a time, мы аккумулируем IDs и выполняем один batched query. Это O(N) → O(1) per level.""Meta's TAO system batches all data fetches within a request. DGS DataLoaders replicate this: instead of one entity at a time, we accumulate IDs and issue one batched query. This turns O(N) → O(1) per level."

3. Mutation Payloads (не исключения)Mutation Payloads (Not Exceptions)

"Почему не throw exceptions для validation?""Why not throw exceptions for validation?" — "GraphQL errors — для infrastructure problems. Business errors (validation, not found) — expected outcomes. Возвращаем их в Payload → client handles с type safety.""GraphQL errors are for infrastructure problems. Business errors (validation, not found) are expected outcomes. Returning them in Payload lets client handle with type safety."

4. Viewer-scoped data (Privacy)Viewer-Scoped Data (Privacy)

"Как вы предотвращаете data leaks?""How do you prevent data leaks?" — "Каждый service method берёт ViewerContext. Каждый SQL query включает WHERE user_id = :viewerId. Defense in depth — даже если resolver забыл проверку, repository не выдаст чужие данные.""Every service method takes ViewerContext. Every SQL query includes WHERE user_id = :viewerId. Defense in depth — even if resolver forgets, repository won't return foreign data."

Вопросы и ответы для интервью Interview Q&A

Q: "Пройдите через request от браузера до БД и обратно"Q: "Walk me through a request from browser to database and back"

A:A: "Браузер отправляет POST в /graphql с query hash (persisted queries) и variables. Spring Security валидирует JWT. DGS смотрит query text по hash. Корневой resolver извлекает ViewerContext из SecurityContext. Сервис проверяет Redis кэш. На cache miss — запрашивает PostgreSQL через Spring Data JPA с WHERE user_id в условии. Для child fields (book.readingSessions) — DataLoader батчит все 20 book IDs и выполняет один SQL. Response нормализуется Relay store (keyed by node ID) и перерендерятся только измененные компоненты.""Browser sends POST to /graphql with persisted query hash and variables. Spring Security validates JWT. DGS looks up query text by hash. Root resolver extracts ViewerContext. Service checks Redis cache first. On miss, queries PostgreSQL with WHERE user_id scoped. For child fields, DataLoader batches all IDs and executes one SQL. Response is normalized into Relay store and only changed components re-render."

Q: "Как бы вы добавили новый widget type?"Q: "How would you add a new widget type?"

A:A: "Три шага, нулевых изменений в existing коде: (1) Backend: новый package с Entity, Repository, Service, Resolver, DataLoader. Реализуем WidgetProvider interface. Spring auto-discovers. (2) GraphQL: добавляем .graphqls с types. Добавляем в WidgetData union в dashboard.graphqls. Запускаем codegen. (3) Frontend: создаём feature folder с Relay fragments и компонентами. Регистрируем в WidgetRegistry.ts (одна строка). Dashboard shell рендерит widget instance из user_dashboard_widget table — works!""Three steps, zero changes: (1) Backend: create package with Entity, Repository, Service, Resolver, DataLoader implementing WidgetProvider. Spring auto-discovers. (2) GraphQL: add .graphqls, add to WidgetData union, run codegen. (3) Frontend: create feature folder, register in WidgetRegistry.ts (one line). Dashboard renders from user_dashboard_widget table — it works."

Q: "Что если Redis упадёт?"Q: "What happens when Redis goes down?"

A:A: "Graceful degradation. Каждый Redis call в try/catch. На failure — fallback к PostgreSQL. Медленнее (500ms vs 100ms) но функционирует. WebSocket notifications — persisted в PostgreSQL, fetched на next load. Eventual consistency вместо real-time.""Graceful degradation. Every Redis call is wrapped in try/catch. On failure, fall back to PostgreSQL directly. Slower (500ms vs 100ms) but works. Notifications persist in DB, fetched on next load."

Q: "Почему cursor-pagination а не offset?"Q: "Why cursor-based pagination instead of offset?"

A:A: "Три причины: (1) Performance: OFFSET 10000 читает и отбрасывает 10K строк. Cursor использует B-Tree seek — O(log n) vs O(n). (2) Consistency: если добавляется книга во время page 2, offset может показать дубликат. Cursor стабилен. (3) Relay native: @connection и usePaginationFragment спроектированы для cursor pagination.""Three reasons: (1) Performance: OFFSET 10000 reads and discards 10K rows. Cursor uses B-Tree seek — O(log n) vs O(n). (2) Consistency: if a book is added during page 2, offset might show duplicates. Cursor is stable. (3) Relay native: @connection and usePaginationFragment are designed for cursor-based pagination."
