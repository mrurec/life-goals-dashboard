Mobile Distribution: публикация LGD в App Store и Play StoreMobile Distribution: shipping LGD to the App Store and Play Store

Зачем отдельный модуль

Разработка RN-клиента (модуль 18) ≠ публикация. Подписание кода, TestFlight, Play Console, скриншоты, политика приватности, review workflow — это целый пласт инфры, который отсутствует в веб-деплое на Vercel. Этот модуль закрывает фазу 5 M8: задачи #119–#122.

Задача не в том, чтобы «собрать релиз один раз», а в том, чтобы push на tag автоматически публиковал bump в TestFlight / Internal Testing без ручных шагов. Это требует secrets-менеджмента, code signing через match, fastlane lanes и GitHub Actions matrix workflow.

Why a separate module

Building the RN client (Module 18) is not the same as shipping it. Code signing, TestFlight, Play Console, screenshots, privacy policy, review workflow — this is a whole layer of infra that doesn't exist in a Vercel web deploy. This module closes M8 Phase 5: tasks #119–#122.

The goal is not to "cut a release once" but to push a tag and have TestFlight / Internal Testing updated automatically, with no manual steps. That requires secrets management, match-based code signing, fastlane lanes, and a GitHub Actions matrix workflow.

fastlane 101

fastlane — Ruby-тулчейн от Google, который унифицирует сборку, подпись и загрузку iOS + Android билдов. Основная единица — lane (цепочка action'ов). LGD использует 4 lane: ios beta, ios release, android beta, android release.

fastlane 101

fastlane is Google's Ruby toolchain that unifies building, signing, and uploading iOS + Android builds. The core unit is a lane (a chain of actions). LGD uses 4 lanes: ios beta, ios release, android beta, android release.

# apps/mobile/fastlane/Fastfile
default_platform(:ios)

platform :ios do
  desc "Build + upload to TestFlight"
  lane :beta do
    match(type: "appstore", readonly: is_ci)        # #119 code signing
    increment_build_number(xcodeproj: "LifeGoals.xcodeproj")
    build_app(scheme: "LifeGoals", export_method: "app-store")
    upload_to_testflight(
      skip_waiting_for_build_processing: true,
      apple_id: ENV["APPLE_ID"],
      api_key_path: "fastlane/api_key.json"
    )
  end
end

platform :android do
  desc "Build + upload to Play Console Internal Testing"
  lane :beta do
    gradle(task: "bundle", build_type: "Release")   # #120
    upload_to_play_store(
      track: "internal",
      aab: "android/app/build/outputs/bundle/release/app-release.aab",
      skip_upload_apk: true,
      json_key: ENV["PLAY_STORE_JSON_KEY"]
    )
  end
end

iOS: match + TestFlight (#119)

fastlane match — канонический способ хранить и раздавать сертификаты/провижины между разработчиками и CI. Сертификаты шифруются и коммитятся в отдельный приватный Git-репо (например, mrurec/life-goals-certificates), расшифровываются на машине только при наличии MATCH_PASSWORD.

Поток публикации: tag mobile-v0.3.0 → CI job на macos-latest → bundle exec fastlane ios beta → TestFlight build processing → email тестерам через 10-15 минут.

iOS: match + TestFlight (#119)

fastlane match is the canonical way to store and share certificates/provisioning profiles across developers and CI. Certificates are encrypted and committed into a separate private Git repo (e.g. mrurec/life-goals-certificates) and are only decrypted with the MATCH_PASSWORD secret.

Release flow: tag mobile-v0.3.0 → macos-latest CI job → bundle exec fastlane ios beta → TestFlight processes the build → testers get an email 10–15 minutes later.

Android: supply + Play Console (#120)

Android проще, чем iOS: один upload keystore (генерируется 1 раз, хранится как base64 в GH secret) + Play Console service account JSON. fastlane supply (подмодуль upload_to_play_store) загружает .aab в Internal Testing track.

Треки Play Console: internal (до 100 тестеров, немедленно) → closed (alpha/beta, до 10000) → open (публичная бета) → production. LGD автоматизирует только internal; промоушены делаются вручную после ручного QA.

Android: supply + Play Console (#120)

Android is simpler than iOS: one upload keystore (generated once, stored as base64 in a GH secret) + a Play Console service account JSON. fastlane supply (alias upload_to_play_store) uploads the .aab to the Internal Testing track.

Play Console tracks: internal (up to 100 testers, instant) → closed (alpha/beta, up to 10 000) → open (public beta) → production. LGD only automates internal; promotions happen manually after manual QA.

# Generate the upload keystore once:
keytool -genkey -v -keystore upload.keystore -alias lifegoals \
  -keyalg RSA -keysize 2048 -validity 10000

# Encode for GitHub secret:
base64 -i upload.keystore | pbcopy

GitHub Actions matrix workflow (#121)

Матрица по платформам: ios на macos-latest (нужен Xcode), android на ubuntu-latest. Кэш: CocoaPods (~/Library/Caches/CocoaPods), Gradle (~/.gradle/caches), Metro (node_modules/.cache), Ruby gems (vendor/bundle).

GitHub Actions matrix workflow (#121)

Matrix by platform: ios on macos-latest (Xcode required), android on ubuntu-latest. Caches: CocoaPods (~/Library/Caches/CocoaPods), Gradle (~/.gradle/caches), Metro (node_modules/.cache), Ruby gems (vendor/bundle).

# .github/workflows/mobile.yml
name: Mobile build + distribute
on:
  push:
    tags: ['mobile-v*']
jobs:
  distribute:
    strategy:
      matrix:
        include:
          - os: macos-latest
            platform: ios
            lane: 'ios beta'
          - os: ubuntu-latest
            platform: android
            lane: 'android beta'
    runs-on: ${{ matrix.os }}
    defaults: { run: { working-directory: apps/mobile } }
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '22', cache: 'npm' }
      - run: npm ci
      - name: Cache Pods
        if: matrix.platform == 'ios'
        uses: actions/cache@v4
        with:
          path: apps/mobile/ios/Pods
          key: ${{ runner.os }}-pods-${{ hashFiles('apps/mobile/ios/Podfile.lock') }}
      - name: Cache Gradle
        if: matrix.platform == 'android'
        uses: actions/cache@v4
        with:
          path: ~/.gradle/caches
          key: ${{ runner.os }}-gradle-${{ hashFiles('apps/mobile/android/**/*.gradle*') }}
      - uses: ruby/setup-ruby@v1
        with: { bundler-cache: true, working-directory: apps/mobile }
      - name: Run fastlane
        env:
          MATCH_PASSWORD: ${{ secrets.MATCH_PASSWORD }}
          APP_STORE_CONNECT_KEY: ${{ secrets.APP_STORE_CONNECT_KEY }}
          PLAY_STORE_JSON_KEY: ${{ secrets.PLAY_STORE_JSON_KEY }}
          ANDROID_KEYSTORE_BASE64: ${{ secrets.ANDROID_KEYSTORE_BASE64 }}
        run: bundle exec fastlane ${{ matrix.lane }}

Store metadata (#122)

Скриншоты, privacy policy, app icon, описание, категория — всё управляется через fastlane/metadata/ (iOS) и fastlane/metadata/android/. Каждый локаль — отдельная подпапка. Скриншоты генерируются автоматически через fastlane snapshot (iOS, использует UI-тесты) или Maestro.

  • Privacy policy — обязательный URL, размещается на lifegoals.app/privacy. Должна честно перечислить, что приложение читает Keychain, FCM-токен, камеру.
  • App Privacy (iOS) + Data safety (Android) — декларативные формы в сторе. Должны соответствовать фактическому сбору данных (widget config, device token, no third-party trackers).
  • Screenshots — минимум 3 размера для iOS (6.9″, 6.5″, 5.5″), 2 для Android (phone + 7″ tablet).
  • Feature graphic — Android-only, 1024×500.

Store metadata (#122)

Screenshots, privacy policy, app icon, description, category — everything is managed via fastlane/metadata/ (iOS) and fastlane/metadata/android/. Each locale is its own subfolder. Screenshots are auto-generated via fastlane snapshot (iOS, backed by UI tests) or Maestro.

  • Privacy policy — mandatory URL, hosted at lifegoals.app/privacy. Must honestly list that the app reads Keychain, FCM token, camera.
  • App Privacy (iOS) + Data safety (Android) — declarative store forms. Must match actual data collection (widget config, device token, no third-party trackers).
  • Screenshots — at least 3 sizes for iOS (6.9″, 6.5″, 5.5″), 2 for Android (phone + 7″ tablet).
  • Feature graphic — Android-only, 1024×500.
