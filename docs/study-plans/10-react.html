Собственные HooksCustom Hooks

Можете написать свой hook, чтобы переиспользовать логику. Это просто функция, которая вызывает другие hooks. You can write your own hook to reuse logic. It's just a function that calls other hooks. 


// useDebounce.ts — custom hook
import { useState, useEffect } from 'react';

export function useDebounce(value: T, delay: number): T {
    const [debouncedValue, setDebouncedValue] = useState(value);

    useEffect(() => {
        const handler = setTimeout(() => {
            setDebouncedValue(value);
        }, delay);

        return () => clearTimeout(handler);
    }, [value, delay]);

    return debouncedValue;
}

// Usage
const SearchComponent: React.FC = () => {
    const [input, setInput] = useState('');
    const debouncedInput = useDebounce(input, 500); // 500ms delay

    useEffect(() => {
        // This effect runs only when debouncedInput changed
        console.log('Searching for:', debouncedInput);
    }, [debouncedInput]);

    return (
         setInput(e.target.value)}
            placeholder="Search..."
        />
    );
};

Что такое Error Boundary?What is Error Boundary?

Error Boundary ловит React ошибки и показывает fallback UI вместо краха приложения. Это class-компонент (не function). Error Boundary catches React errors and shows fallback UI instead of crashing the app. It's a class component (not function). 


// ErrorBoundary.tsx
import React, ReactNode, ReactElement from 'react';

interface ErrorBoundaryProps {
    children: ReactNode;
    fallback?: ReactElement;
}

interface ErrorBoundaryState {
    hasError: boolean;
    error?: Error;
}

class ErrorBoundary extends React.Component {
    constructor(props: ErrorBoundaryProps) {
        super(props);
        this.state = { hasError: false };
    }

    static getDerivedStateFromError(error: Error): ErrorBoundaryState {
        return { hasError: true, error };
    }

    componentDidCatch(error: Error, errorInfo: React.ErrorInfo) {
        console.error('Error caught:', error, errorInfo);
    }

    render() {
        if (this.state.hasError) {
            return this.props.fallback || 
Something went wrong
;
        }
        return this.props.children;
    }
}

export default ErrorBoundary;

Что нового в React 19What's New in React 19

React 19.2 привносит мощные возможности для работы с формами, асинхронными операциями и оптимизацией UI. Все эти паттерны нативны в React и не требуют дополнительных библиотек. React 19.2 brings powerful capabilities for forms, async operations, and UI optimization. All these patterns are native to React and don't require additional libraries.

useActionState — Асинхронные формыuseActionState — Async Forms

`useActionState` — новый hook для обработки асинхронных операций в формах. Автоматически управляет pending, error, и data состояниями. Идеально для GraphQL мутаций. `useActionState` — new hook for handling async operations in forms. Automatically manages pending, error, and data states. Perfect for GraphQL mutations.

useOptimistic — Optimistic UI UpdatesuseOptimistic — Optimistic UI Updates

Обновляет UI мгновенно, пока запрос обрабатывается. Если сервер вернёт ошибку, UI откатится. Отлично для real-time feel: отметить задачу как решённую, и сразу видно в UI. Updates UI immediately while request processes. If server returns error, UI rolls back. Great for real-time feel: mark problem solved and see it immediately in UI.

Activity API — Сохранение состояния скрытых компонентовActivity API — Preserve Hidden Component State

`<Activity mode="hidden">` заменил Offscreen API. Позволяет скрыть компонент, но сохранить его состояние и скролл. Полезно для tab-like интерфейсов, где нужна мгновенная смена без потери контекста. `<Activity mode="hidden">` replaces Offscreen API. Allows hiding a component while preserving its state and scroll. Useful for tab-like interfaces where you need instant switching without losing context.

Server Components (RSC Pattern)Server Components (RSC Pattern)

Server Components выполняются на сервере и отправляют в браузер только отрендеренный HTML + JavaScript для интерактивности. Основная польза: доступ к БД на сервере, безопасные secrets, меньше JavaScript в браузере. Server Components run on the server and send only rendered HTML + JavaScript for interactivity to the browser. Key benefits: database access on server, safe secrets, less JavaScript in browser.

ref как prop — без forwardRefref as prop — without forwardRef

В React 19 ref автоматически передаётся как prop. Больше не нужен `forwardRef()`. In React 19, ref is automatically passed as a prop. No more need for `forwardRef()`.

Document Metadata в JSXDocument Metadata in JSX

Теперь можно писать `<title>`, `<meta>` прямо в компоненте. React автоматически поместит их в `<head>`. Now you can write `<title>`, `<meta>` directly in components. React automatically places them in `<head>`.

Batched Suspense RevealBatched Suspense Reveal

Если несколько компонентов разрешат Suspense примерно в один момент (благодаря @defer в GraphQL), они отобразятся вместе, не мигая по отдельности. If multiple components resolve Suspense around the same time (thanks to @defer in GraphQL), they reveal together without separate flickers.
