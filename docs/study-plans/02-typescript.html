Type vs Interface (День 1-2)Type vs Interface (Day 1-2)

Type используется для скалярных значений, объединений и интерсекций. Interface используется для структур объектов и расширяемости.

Type is used for scalar values, unions, and intersections. Interface is used for object structures and extensibility. 


// Type для скалярного значения
type WidgetType = "INTERVIEW_PREP" | "FITNESS" | "BUDGET";

// Type для объекта
type FoodProductType = {
    id: string;
    name: string;
    caloriesPer100g: number;
    isCustom?: boolean;  // опциональное поле
};

// Interface для объекта (более традиционный для структур)
interface FoodProduct {
    id: string;
    name: string;
    caloriesPer100g: number;
    isCustom?: boolean;
}

// Interface может быть расширен (extends)
interface CustomFoodProduct extends FoodProduct {
    userId: string;
    createdAt: Date;
}

// Type может использовать интерсекцию (&)
type FullFoodProduct = FoodProduct & {
    userId: string;
    createdAt: Date;
};

// Interface может быть объявлена несколько раз (merging)
interface Window {
    myCustomProperty: string;
}

interface Window {
    anotherProperty: number;
}
// Обе декларации объединены в одну

// Type алиас для функции
type ValidationFn = (value: unknown) => boolean;

const isValidEmail: ValidationFn = (value) => {
    return typeof value === "string" && value.includes("@");
};
Best Practice для LGD: Используйте `interface` для структур объектов (FoodProduct, Recipe, Workout), а `type` для union типов (WidgetType, HttpMethod). Это облегчает расширяемость.
Best Practice for LGD: Use `interface` for object structures (FoodProduct, Recipe, Workout), and `type` for union types (WidgetType, HttpMethod). This improves extensibility.

Generics (День 2-3)Generics (Day 2-3)

Основные концепцииBasic Concepts

Generics позволяют писать код, который работает с любыми типами, но сохраняет типизацию. В LGD они используются в Relay Connection паттерне и кастомных хуках.

Generics allow you to write code that works with any type while maintaining type safety. In LGD they are used in Relay Connection pattern and custom hooks. 


// Generic функция
function getFirstItem<T>(array: T[]): T | undefined {
    return array[0];
}

const first = getFirstItem<string>(["a", "b"]);  // string
const num = getFirstItem<number>([1, 2]);        // number

// Generic interface
interface Container<T> {
    value: T;
    getValue(): T;
}

interface StringContainer extends Container<string> {
    toUpperCase(): string;
}

// Generic type для Relay Connection pattern
interface Connection<T> {
    edges: Array<{
        node: T;
        cursor: string;
    }>;
    pageInfo: {
        hasNextPage: boolean;
        endCursor: string | null;
    };
    totalCount: number;
}

// Использование
interface CodingProblem {
    id: string;
    title: string;
    difficulty: "EASY" | "MEDIUM" | "HARD";
}

const problemConnection: Connection<CodingProblem> = {
    edges: [
        {
            node: { id: "1", title: "Array Sum", difficulty: "EASY" },
            cursor: "cursor1"
        }
    ],
    pageInfo: {
        hasNextPage: true,
        endCursor: "cursor1"
    },
    totalCount: 100
};

Constraints (ограничения)Constraints


// Generic с constraint: T должно иметь свойство 'id'
interface HasId {
    id: string;
}

function getId<T extends HasId>(obj: T): string {
    return obj.id;
}

getId({ id: "123", name: "John" });  // OK
// getId({ name: "John" });  // Ошибка: нет 'id'

// Constraint с keyof
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
    return obj[key];
}

const person = { name: "Alice", age: 30 };
const name = getProperty(person, "name");  // OK, тип string
// const invalid = getProperty(person, "email");  // Ошибка: 'email' не существует

// Constraint с множественным наследованием
function merge<T extends object, U extends object>(obj1: T, obj2: U): T & U {
    return { ...obj1, ...obj2 };
}

const merged = merge({ a: 1 }, { b: 2 });
// merged.a и merged.b доступны, типизировано

Conditional Types (продвинутый уровень)Conditional Types (Advanced)


// Conditional type: если T является строкой, вернуть number, иначе string
type StringToNumber<T> = T extends string ? number : string;

type A = StringToNumber<string>;  // number
type B = StringToNumber<boolean>;  // string

// Используется для типобезопасных утилит
type Flatten<T> = T extends Array<infer U> ? U : T;

type Str = Flatten<string[]>;      // string
type Num = Flatten<number>;        // number

Utility Types (День 3)Utility Types (Day 3)

TypeScript поставляется с встроенными utility типами, которые облегчают манипуляцию типами:

TypeScript comes with built-in utility types that make type manipulation easier:

UtilityUtility ОписаниеDescription ПримерExample
Partial<T> Делает все поля опциональнымиMakes all fields optional Partial<FoodProduct>
Required<T> Делает все поля обязательнымиMakes all fields required Required<FoodProduct>
Pick<T, K> Выбирает определённые поля из TSelects specific fields from T Pick<FoodProduct, "name" | "id">
Omit<T, K> Исключает определённые поля из TExcludes specific fields from T Omit<FoodProduct, "isCustom">
Record<K, T> Объект с ключами K и значениями TObject with keys K and values T Record<"morning" | "evening", number>
Readonly<T> Делает все поля read-onlyMakes all fields read-only Readonly<FoodProduct>
ReturnType<T> Извлекает тип возвращаемого значения функцииExtracts function return type ReturnType<typeof fetchProducts>

Примеры использованияUsage Examples


interface FoodProduct {
    id: string;
    name: string;
    caloriesPer100g: number;
    isCustom?: boolean;
}

// Partial: для patch-запроса (обновление некоторых полей)
function updateProduct(id: string, changes: Partial<FoodProduct>): void {
    // changes может содержать только некоторые поля
}

// Pick: для DTO (Data Transfer Object) — отправляем только нужные данные
type ProductPreview = Pick<FoodProduct, "id" | "name">;
const preview: ProductPreview = { id: "1", name: "Apple" };

// Omit: исключаем чувствительные поля
type PublicProduct = Omit<FoodProduct, "isCustom">;

// Record: словарь макро по типам
type MacroSplit = Record<"protein" | "carbs" | "fat", number>;
const macros: MacroSplit = {
    protein: 30,
    carbs: 40,
    fat: 30
};

// Readonly: неизменяемые константы
const DIFFICULTY_LEVELS: Readonly<string[]> = ["EASY", "MEDIUM", "HARD"];
// DIFFICULTY_LEVELS[0] = "INSANE";  // Ошибка!

// ReturnType: автоматический тип результата функции

    return Promise.resolve([]);

type ProblemsResult = ReturnType<typeof fetchProblems>;  // Promise<FoodProduct[]>

Stricter checkJS (JavaScript файлы с контролем типов)

TypeScript 6 по умолчанию더 строго проверяет JavaScript файлы с JSDoc комментариями. Это помогает постепенно мигрировать JS проекты на TypeScript без переписывания всего кода.

TypeScript 6 теперь ESM-first, что означает лучшую совместимость с modern bundler'ами (Vite, Rolldown). Все импорты по умолчанию рассматриваются как ES modules, повышая производительность сборки.

Улучшенная производительность

Компилятор переписан для скорости. Проверка типов на больших проектах (10K+ файлов) теперь быстрее на 20-30%. Это заметно при разработке Life Goals Dashboard с 200+ компонентами.

Улучшенный `satisfies` оператор

satisfies оператор теперь работает более предсказуемо с union типами и object shorthand. Это полезно для конфигурации виджетов, где нужна гибкость типа и проверка структуры одновременно.

describe('Backlog #2: Frontend Project Structure', () => {
  const rootDir = path.resolve(__dirname, '..');

  it('should have tsconfig.json with strict mode enabled', () => {
    const tsconfig = JSON.parse(
      fs.readFileSync(path.join(rootDir, 'tsconfig.json'), 'utf-8')
    );
    expect(tsconfig.compilerOptions.strict).toBe(true);
  });

  it('should target ESNext for modern React', () => {
    const tsconfig = JSON.parse(
      fs.readFileSync(path.join(rootDir, 'tsconfig.json'), 'utf-8')
    );
    expect(tsconfig.compilerOptions.target).toMatch(/ESNext|ES2022/i);
  });

  it('should have jsx set to react-jsx', () => {
    const tsconfig = JSON.parse(
      fs.readFileSync(path.join(rootDir, 'tsconfig.json'), 'utf-8')
    );
    expect(tsconfig.compilerOptions.jsx).toBe('react-jsx');
  });

  it('should have required source directories', () => {
    const requiredDirs = [
      'src/components',
      'src/features',
      'src/hooks',
      'src/atoms',
      'src/relay',
      'src/styles',
    ];
    for (const dir of requiredDirs) {
      expect(
        fs.existsSync(path.join(rootDir, dir))
      ).toBe(true);
    }
  });

  it('should have proper type definitions', () => {
    // vite-env.d.ts should declare CSS Modules
    const envDts = fs.readFileSync(
      path.join(rootDir, 'src', 'vite-env.d.ts'), 'utf-8'
    );
    expect(envDts).toContain('.module.css');
  });
});
