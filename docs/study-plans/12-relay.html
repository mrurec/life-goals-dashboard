Введение: Relay для управления GraphQL данными

Relay — это мощная библиотека для управления GraphQL данными на фронтенде. Её главный принцип: Fragment Colocation — каждый компонент объявляет свой GraphQL фрагмент в том же файле.

За 4 дня вы освоите:

  • Fragment Colocation: `ComponentName_fieldName` в том же файле .tsx
  • useLazyLoadQuery для инициализации данных на странице
  • usePaginationFragment для Relay Connection pagination (cursor-based)
  • useMutation для отправки данных и оптимистических обновлений
  • Environment setup: Network, Store, RecordSource, WebSocket subscriptions
  • Relay Compiler: `npm run relay` генерирует __generated__ типы
Совет: Relay требует рефакторинга мышления. Вместо "fetch all data, then render", вы объявляете requirements в компонентах, и Relay собирает всё в один эффективный запрос. Это похоже на GraphQL-native подход Meta.

Introduction: Relay for GraphQL Data Management

Relay is a powerful library for managing GraphQL data on the frontend. Its core principle: Fragment Colocation — each component declares its own GraphQL fragment in the same file.

Over 4 days you will master:

  • Fragment Colocation: `ComponentName_fieldName` in the same .tsx file
  • useLazyLoadQuery for data initialization on the page
  • usePaginationFragment for Relay Connection pagination (cursor-based)
  • useMutation for sending data and optimistic updates
  • Environment setup: Network, Store, RecordSource, WebSocket subscriptions
  • Relay Compiler: `npm run relay` generates __generated__ types
Tip: Relay requires a shift in thinking. Instead of "fetch all data, then render", you declare requirements in components, and Relay gathers everything into one efficient query. This is similar to Meta's GraphQL-native approach.

Fragment Colocation — главный принцип

Что такое Fragment?

Fragment — это именованный набор полей GraphQL. Компонент объявляет, какие данные ему нужны. Parent компонент собирает фрагменты и строит полный запрос.

WorkoutCard.tsx — Example с Fragment


import { Fragment, useFragment } from 'react-relay';
import { graphql } from 'relay-runtime';
import type { WorkoutCard_workout$key } from './__generated__/WorkoutCard_workout.graphql';

interface WorkoutCardProps {
    workoutRef: WorkoutCard_workout$key;
}

// REQUIRED pattern: Fragment IN THE SAME FILE with the component
const WorkoutCard: React.FC = ({ workoutRef }) => {
    // useFragment fetches data from the Relay store
    const workout = useFragment(
        graphql`
            fragment WorkoutCard_workout on Workout {
                id
                exerciseName
                durationMinutes
                caloriesBurned
                createdAt
            }
        `,
        workoutRef
    );

    return (
        

            
{workout.exerciseName}

            
Duration: {workout.durationMinutes} min

            
Calories: {workout.caloriesBurned}

            
Date: {new Date(workout.createdAt).toLocaleDateString()}

        

    );
};

export default WorkoutCard;

Правила Fragment Colocation

  • Имя фрагмента: `ComponentName_fieldName`
  • Местоположение: ВО ВСЕ М ФАЙЛЕ компонента (.tsx)
  • Данные: Компонент описывает только данные, которые ему нужны
  • Типы: Relay генерирует типы в `__generated__/ComponentName_fieldName.graphql.ts`
КРИТИЧНО: Никогда не создавайте отдельный файл для фрагмента. Фрагмент ВСЕГДА в том же файле, что и компонент. Это ключ к поддерживаемости.

Fragment Colocation Rules

  • Fragment name: `ComponentName_fieldName`
  • Location: IN THE SAME FILE as the component (.tsx)
  • Data: The component only describes the data it needs
  • Types: Relay generates types in `__generated__/ComponentName_fieldName.graphql.ts`
CRITICAL: Never create a separate file for a fragment. The fragment is ALWAYS in the same file as the component. This is the key to maintainability.

Mutations: useMutation и Optimistic Updates

Mutations: useMutation and Optimistic Updates

Отправка данных на сервер

Sending Data to the Server


import { useMutation, graphql } from 'react-relay';
import type { LogWorkoutMutation } from './__generated__/LogWorkoutMutation.graphql';

interface LogWorkoutFormProps {
    onWorkoutLogged?: () => void;
}

const LogWorkoutForm: React.FC = ({ onWorkoutLogged }) => {
    const [exerciseName, setExerciseName] = useState('');
    const [duration, setDuration] = useState('');

    const [commitMutation, isMutationInFlight] = useMutation(
        graphql`
            mutation LogWorkoutMutation($input: LogWorkoutInput!) {
                logWorkout(input: $input) {
                    workout {
                        id
                        exerciseName
                        durationMinutes
                    }
                    errors {
                        field
                        message
                        code
                    }
                }
            }
        `
    );

    const handleSubmit = (e: React.FormEvent) => {
        e.preventDefault();

        commitMutation({
            variables: {
                input: {
                    exerciseName,
                    durationMinutes: parseInt(duration, 10),
                    caloriesBurned: 0
                }
            },
            onCompleted: (response) => {
                // Check for errors in the payload
                if (response.logWorkout.errors.length > 0) {
                    response.logWorkout.errors.forEach(error => {
                        console.error(`${error.field}: ${error.message}`);
                    });
                } else {
                    console.log('Workout logged:', response.logWorkout.workout);
                    setExerciseName('');
                    setDuration('');
                    onWorkoutLogged?.();
                }
            },
            onError: (error) => {
                console.error('Network error:', error);
            }
        });
    };

    return (
        

             setExerciseName(e.target.value)}
                placeholder="Exercise name"
                required
            />
             setDuration(e.target.value)}
                placeholder="Duration (minutes)"
                required
            />
            
        

    );
};

export default LogWorkoutForm;

Optimistic Updates

Optimistic Updates

Оптимистические обновления показывают результат пользователю сразу, до ответа сервера. Если есть ошибка, Relay откатывает изменения.
Optimistic updates show the result to the user immediately, before the server responds. If there is an error, Relay rolls back the changes.


const handleSubmit = () => {
    commitMutation({
        variables: { input },
        // Optimistically update UI BEFORE server response
        optimisticResponse: {
            logWorkout: {
                __typename: 'LogWorkoutPayload',
                workout: {
                    id: 'temp-id-' + Date.now(),
                    exerciseName: input.exerciseName,
                    durationMinutes: input.durationMinutes,
                },
                errors: []
            }
        },
        // Update Relay store
        updater: (store) => {
            const myWorkouts = store
                .getRoot()
                .getLinkedRecords('myWorkouts');

            if (myWorkouts) {
                const newWorkout = store.getRootField('logWorkout')?.getLinkedRecord('workout');
                if (newWorkout) {
                    myWorkouts.push(newWorkout);
                }
            }
        },
        onCompleted: (response) => {
            // Check the result
        }
    });
};

Environment Setup: Network и Store

Environment Setup: Network and Store

RelayEnvironment.ts

RelayEnvironment.ts


import {
    Environment,
    Network,
    RecordSource,
    Store,
    SubscriptionRoot,
} from 'relay-runtime';
import { createClient } from 'graphql-ws';

// Network layer: how to send requests to the server
const fetchFn = async (params: any, variables: any) => {
    const response = await fetch('http://localhost:8080/graphql', {
        method: 'POST',
        headers: {
            'Content-Type': 'application/json',
            'Authorization': `Bearer ${localStorage.getItem('auth_token')}`
        },
        body: JSON.stringify({
            query: params.text,
            variables
        })
    });

    const json = await response.json();

    if (!response.ok) {
        throw new Error(`GraphQL error: ${json.errors?.[0]?.message}`);
    }

    return json;
};

// WebSocket for subscriptions
const wsClient = createClient({
    url: 'ws://localhost:8080/subscriptions',
    connectionParams: {
        authorization: `Bearer ${localStorage.getItem('auth_token')}`
    }
});

const subscriptionsFn = (operation: any, variables: any) => {
    return {
        subscribe: (sink: any) => {
            const dispose = wsClient.subscribe(
                {
                    operationName: operation.name,
                    query: operation.text,
                    variables
                },
                sink
            );
            return dispose;
        }
    };
};

// Create Environment
const environment = new Environment({
    network: Network.create(fetchFn, subscriptionsFn),
    store: new Store(new RecordSource())
});

export default environment;

Как работает Relay Compiler

How Relay Compiler Works

  1. Сканирует все `.ts` и `.tsx` файлы в `src/`
  2. Находит все `graphql` теги (фрагменты, запросы, мутации)
  3. Генерирует TypeScript типы в `__generated__/` папках
  4. Компилирует GraphQL в JavaScript runtime операции

После запуска `npm run relay` вы увидите папки `__generated__/` рядом с вашими компонентами. Не редактируйте эти файлы вручную — это автогенерированный код.
After running `npm run relay` you will see `__generated__/` folders next to your components. Do not edit these files manually — this is auto-generated code.

      expect(environment.mock.getAllOperations().length).toBeGreaterThan(0);

      </RelayEnvironmentProvider>
    );

    const input = screen.getByPlaceholderText(/search|поиск/i);
    fireEvent.change(input, { target: { value: 'chicken' } });
    jest.advanceTimersByTime(300);

            caloriesPer100g: 165,
            proteinPer100g: 31,

          };

    );

    await waitFor(() => {

      expect(screen.getByText(/165/)).toBeInTheDocument(); // calories
