Введение: зачем React Native для LGD

Milestone 8 — мобильное приложение apps/mobile. Оно разделяет GraphQL-схему, Jotai-атомы, hooks и типы с веб-клиентом через packages/shared. Всё, что завязано на DOM (document, window, localStorage, CSS Modules), остаётся на вебе. Всё, что завязано на нативные API (AsyncStorage, Keychain, FCM, StyleSheet), живёт в apps/mobile.

RN использует тот же React 19.2, те же хуки, те же фрагменты Relay — отличие только в примитивах (View вместо div) и бандлере (Metro вместо Vite). Для Meta/FAANG-портфолио это даёт важный сигнал: автор умеет проектировать cross-platform monorepo, а не просто писать SPA.

Этот модуль проходится после модулей 10 (React), 12 (Relay) и 13 (Jotai) — это база. Здесь вы закрываете 21 задачу backlog'а (#98–#118, все фазы M8 кроме distribution).

Introduction: Why React Native for LGD

Milestone 8 introduces apps/mobile — a React Native client. It shares the GraphQL schema, Jotai atoms, hooks, and types with the web client through packages/shared. Everything DOM-bound (document, window, localStorage, CSS Modules) stays web-only. Everything platform-bound (AsyncStorage, Keychain, FCM, StyleSheet) lives under apps/mobile.

RN uses the same React 19.2, the same hooks, the same Relay fragments — the only differences are the primitives (View instead of div) and the bundler (Metro instead of Vite). For a Meta/FAANG portfolio this is a strong signal that the author can design a cross-platform monorepo, not just a SPA.

Take this module after Module 10 (React), Module 12 (Relay), and Module 13 (Jotai). It closes 21 backlog tasks (#98–#118 — every M8 phase except distribution).

Relay + WebSocket (#100)

Мобильный RelayEnvironment использует createNetwork(transport) из packages/shared/src/relay/ (задача #107). Транспорт для мутаций/запросов — global.fetch, для подписок — graphql-ws через нативный WebSocket.

Relay + WebSocket (#100)

The mobile RelayEnvironment consumes createNetwork(transport) from packages/shared/src/relay/ (task #107). The transport is global.fetch for queries/mutations and graphql-ws over a native WebSocket for subscriptions.

// apps/mobile/src/relay/RelayEnvironment.ts
import { Environment, RecordSource, Store } from 'relay-runtime';
import { createClient } from 'graphql-ws';
import { createNetwork } from '@life-goals/shared/relay';
import { getAuthToken } from '../auth/tokenStore'; // Keychain-backed

const wsClient = createClient({
  url: process.env.GRAPHQL_WS_ENDPOINT!,
  connectionParams: async () => ({ authorization: `Bearer ${await getAuthToken()}` }),
});

const network = createNetwork({
  fetchFn: global.fetch,
  subscribeFn: wsClient,
  getAuthHeader: async () => `Bearer ${await getAuthToken()}`,
});

export const environment = new Environment({ network, store: new Store(new RecordSource()) });
Внимание: global.fetch в RN не поддерживает streaming (@defer). Если используете @defer на вебе, то для мобильника нужно собирать полноценный ответ через react-native-fetch-api + react-native-url-polyfill или дождаться fetch-streams в Hermes.
Caveat: RN's global.fetch does not support streaming (@defer). If you rely on @defer on the web, you need react-native-fetch-api + react-native-url-polyfill on mobile, or wait for streaming fetch in Hermes.

OAuth + Keychain (#102)

react-native-app-auth делает PKCE-флоу с Google/GitHub без кастомного backend'а, react-native-keychain хранит JWT в iOS Keychain и Android EncryptedSharedPreferences (аппаратно защищённое хранилище). Это замена localStorage для токенов.

OAuth + Keychain (#102)

react-native-app-auth runs the PKCE flow with Google/GitHub without a custom backend. react-native-keychain stores the JWT in the iOS Keychain and Android EncryptedSharedPreferences (hardware-backed). It replaces localStorage for tokens.

// apps/mobile/src/auth/tokenStore.ts
import * as Keychain from 'react-native-keychain';

const SERVICE = 'life-goals-jwt';

export async function setAuthToken(token: string) {
  await Keychain.setGenericPassword('jwt', token, {
    service: SERVICE,
    accessible: Keychain.ACCESSIBLE.WHEN_UNLOCKED_THIS_DEVICE_ONLY,
  });
}

export async function getAuthToken(): Promise<string | null> {
  const result = await Keychain.getGenericPassword({ service: SERVICE });
  return result ? result.password : null;
}

export async function clearAuthToken() {
  await Keychain.resetGenericPassword({ service: SERVICE });
}
CRITICAL: никогда не держите JWT в AsyncStorage. Это plain file в sandbox приложения и извлекается с rooted/jailbroken устройства за секунды.
CRITICAL: never put a JWT in AsyncStorage. It's a plain file inside the app sandbox and can be extracted from a rooted/jailbroken device in seconds.

Push-уведомления: FCM + APNS (#115)

@react-native-firebase/messaging унифицирует FCM (Android) и APNS (iOS): один API, две прошивки. Сервер отправляет deviceToken, бэкенд связывает его с user_id и шлёт через Firebase Cloud Messaging.

Интеграция с уведомлениями Notification (M5): когда AchievementChecker создаёт in-app нотификацию, тот же сервис параллельно отправляет FCM push. Это даёт resilient delivery — даже если WebSocket-подписка упала, push разбудит пользователя.

Push notifications: FCM + APNS (#115)

@react-native-firebase/messaging unifies FCM (Android) and APNS (iOS): one API, two firmwares. The client registers its deviceToken, the backend links it to a user_id, and dispatches via Firebase Cloud Messaging.

This integrates with the Notification service (M5): whenever AchievementChecker creates an in-app notification, the same service also fires an FCM push. This yields resilient delivery — even if the WebSocket subscription drops, the push still wakes up the user.

// apps/mobile/src/push/registerDevice.ts
import messaging from '@react-native-firebase/messaging';
import { commitMutation, graphql } from 'react-relay';
import { environment } from '../relay/RelayEnvironment';

export async function registerDeviceForPush() {
  const authStatus = await messaging().requestPermission();
  const enabled = authStatus === messaging.AuthorizationStatus.AUTHORIZED;
  if (!enabled) return;

  const deviceToken = await messaging().getToken();
  commitMutation(environment, {
    mutation: graphql`
      mutation registerDeviceForPushMutation($input: RegisterDeviceInput!) {
        registerDevice(input: $input) { device { id } errors { message code } }
      }`,
    variables: { input: { token: deviceToken, platform: Platform.OS.toUpperCase() } },
  });
}

Биометрия (#116)

Face ID / Touch ID / Fingerprint защищают доступ к приложению после блокировки на 5 минут (AppState listener). Fallback — PIN или полный re-login. Проверка биометрии не заменяет JWT, она лишь разблокирует уже сохранённый в Keychain токен.

Biometrics (#116)

Face ID / Touch ID / Fingerprint gate access to the app after a 5-minute idle lock (AppState listener). Fallback: PIN or full re-login. Biometric validation does not replace the JWT — it only unlocks the Keychain-stored token.

Offline-first + mutation queue (#117)

Relay Store хранится в AsyncStorage через relay-runtime-совместимую обёртку. На reconnect — FIFO-очередь мутаций дренируется в фоне. Для UX конфликтов (stale optimistic update vs server winner) используйте updater в mutation config с явной стратегией (server-wins / last-write-wins / manual resolution).

@react-native-community/netinfo даёт сигнал online/offline для интеграции: UI badge «Offline», disabled на некритичных экранах, автосинк при восстановлении сети.

Offline-first + mutation queue (#117)

The Relay store is persisted to AsyncStorage through a relay-runtime-compatible wrapper. On reconnect, a FIFO mutation queue drains in the background. For conflict UX (stale optimistic update vs server winner), use the updater in the mutation config with an explicit strategy (server-wins / last-write-wins / manual resolution).

@react-native-community/netinfo emits the online/offline signal for UI: an "Offline" badge, disabling non-critical screens, and auto-sync when the network returns.
