Less boilerplate (data classes instead of Lombok), type-level null-safety, coroutines, full interop with the Java ecosystem (Spring, Hibernate, Jackson). A deliberate choice aligned with modern FAANG team standards.

Синтаксис и null-safetySyntax & null-safety

Переменные: val и var

val — immutable (неизменяемая). Основной выбор в Kotlin. var — mutable, используется редко (только когда действительно нужно перезаписать).

Variables: val and var

val — immutable. The primary choice in Kotlin. var — mutable, used rarely (only when reassignment is truly needed).

val name = "Kotlin"           // immutable
var counter = 0               // mutable, reassignable
val numbers: List<Int> = listOf(1, 2, 3)

Null-safety — фундамент безопасного backend

Kotlin разделяет nullable (String?) и non-nullable (String) типы на уровне компилятора. NPE становится компиляционной ошибкой, а не runtime-сюрпризом в продакшене.

Null-safety — foundation of safe backend

Kotlin distinguishes nullable (String?) and non-nullable (String) at the compiler level. NPE becomes a compile error, not a runtime surprise in production.

val notNull: String = "Hello"
val nullable: String? = null

// Safe call
val length = nullable?.length ?: 0

// Elvis operator
val value = nullable ?: "default"

// Let-block for nullable
nullable?.let { println("Got value: $it") }
Не использовать !!Avoid !!

Оператор !! (not-null assertion) выбрасывает NPE и обнуляет всю пользу null-safety. В проекте LGD мы его не используем — вместо этого Elvis, requireNotNull() с осмысленным сообщением или sealed-результат.

The !! operator (not-null assertion) throws NPE and defeats the whole point of null-safety. In LGD we don't use it — prefer Elvis, requireNotNull() with a meaningful message, or a sealed result type.

Scope functions: let, run, apply, also, with

Пять функций, которые меняют контекст выполнения блока. Запоминайте по двум осям: что возвращают (объект или результат блока) и как обращаются к объекту (it или this).

Scope functions: let, run, apply, also, with

Five functions that shift the execution context. Remember by two axes: what they return (the object or the block result) and how they reference the object (it or this).

data class User(var name: String, var age: Int)

// apply — configure object, return it
val user = User("Alice", 25).apply {
    name = "Bob"
    age = 30
}

// let — transform nullable / scope result
val nameLength: Int? = user.name.takeIf { it.isNotEmpty() }?.let { it.length }

// also — side effect (logging), return object
val savedUser = userRepository.save(user).also {
    log.info("Saved user ${it.id}")
}

Классы: data / sealed / enumClasses: data / sealed / enum

Регулярные классы

Классы в Kotlin final по умолчанию. Чтобы разрешить наследование — open. Это сознательная противоположность Java и защита от хрупких иерархий.

Regular classes

Classes in Kotlin are final by default. To allow inheritance, mark open. A deliberate inversion of Java that prevents fragile hierarchies.

open class Animal(val name: String) {
    open fun sound() = println("Some sound")
}

class Dog(name: String) : Animal(name) {
    override fun sound() = println("Woof!")
}

Data classes — основа доменных моделей

Data-класс автоматически генерирует equals, hashCode, toString, copy и componentN. В LGD все DTO, payload'ы мутаций, entity-модели — data-классы.

Data classes — the foundation of domain models

A data class auto-generates equals, hashCode, toString, copy, and componentN. In LGD every DTO, mutation payload, and entity model is a data class.

data class Recipe(
    val id: UUID,
    val name: String,
    val ingredients: List<String>,
    val caloriesPer100g: Int
)

val recipe = Recipe(UUID.randomUUID(), "Pasta", listOf("flour", "egg"), 350)
val copied = recipe.copy(name = "Pizza")   // cheap immutable update

Sealed classes — для Result/ValidationResult

Sealed-класс ограничивает подтипы одним файлом/модулем. Идеально для ValidationResult, Result, состояний UI-машины. Компилятор проверяет, что when покрывает все ветки.

Sealed classes — for Result/ValidationResult

A sealed class restricts subtypes to one file/module. Perfect for ValidationResult, Result, UI state machines. The compiler enforces when exhaustiveness.

sealed class ValidationResult {
    data object Success : ValidationResult()
    data class ValidationError(val field: String, val message: String) : ValidationResult()
    data class SystemError(val cause: Throwable) : ValidationResult()
}

fun handle(result: ValidationResult) = when (result) {
    is ValidationResult.Success -> "OK"
    is ValidationResult.ValidationError -> "Invalid ${result.field}: ${result.message}"
    is ValidationResult.SystemError -> "Internal error"
    // no else — compiler enforces exhaustiveness
}

CoroutinesCoroutines

Coroutines — легковесные «зелёные потоки». В DGS v11 резолверы могут быть suspend fun — фреймворк сам маппит их в CompletableFuture. Это позволяет писать асинхронный код как синхронный.

Coroutines are lightweight "green threads". In DGS v11 resolvers can be suspend fun — the framework maps them to CompletableFuture. This lets you write async code as if it were synchronous.

suspend fun fetchUserData(userId: UUID): User {
    delay(100)  // non-blocking
    return userRepository.findById(userId).orElseThrow()
}

// Dispatchers
launch(Dispatchers.IO) {      // blocking I/O
    saveToDisk(data)
}
launch(Dispatchers.Default) { // CPU-heavy work
    computeReadinessScore()
}

// Structured concurrency
coroutineScope {
    val user = async { fetchUser(id) }
    val widgets = async { fetchWidgets(id) }
    Dashboard(user.await(), widgets.await())
}

class SpacedRepetitionServiceTest {

    private val service = SpacedRepetitionService()

    @Test
    fun `high quality answer should increase interval and ease factor`() {
        val previous = ReviewSchedule(
            problemId = UUID.randomUUID(),
            easeFactor = 2.5,
            intervalDays = 3,
            repetitions = 1,
            nextReviewDate = LocalDate.now()

        assertThat(next.repetitions).isEqualTo(2)
        assertThat(next.easeFactor).isGreaterThan(previous.easeFactor)
        assertThat(next.intervalDays).isGreaterThan(previous.intervalDays)

    @Test
    fun `failed answer should reset interval to 1 and keep ease factor floor at 1_3`() {
        val previous = ReviewSchedule(

            easeFactor = 1.35,
            intervalDays = 14,
            repetitions = 5,
            nextReviewDate = LocalDate.now()

        assertThat(next.intervalDays).isEqualTo(1)
        assertThat(next.repetitions).isZero()

A classic Kotlin task: sealed provider hierarchy, Spring DI for auto-discovery, contract tests. Fits perfectly on enum + sealed + null-safe lookup.

Test:

sealed interface AccountProvider {
    val kind: ProviderKind
    suspend fun fetchBalance(credential: Credential): Money
}

enum class ProviderKind { MANUAL, CRYPTO_COM, OPEN_BANKING }

@Component
class AccountProviderRegistry(providers: List<AccountProvider>) {
    private val byKind = providers.associateBy { it.kind }
    fun find(kind: ProviderKind): AccountProvider =
        byKind[kind] ?: error("No provider registered for $kind")
}

class AccountProviderRegistryTest {

    @Test
    fun `registry should resolve provider by kind`() {
        val manual = mockk<AccountProvider> { every { kind } returns ProviderKind.MANUAL }
        val crypto = mockk<AccountProvider> { every { kind } returns ProviderKind.CRYPTO_COM }

        val registry = AccountProviderRegistry(listOf(manual, crypto))

        assertThat(registry.find(ProviderKind.MANUAL)).isSameAs(manual)
        assertThat(registry.find(ProviderKind.CRYPTO_COM)).isSameAs(crypto)
    }

    @Test
    fun `registry should throw when kind is unknown`() {
        val registry = AccountProviderRegistry(emptyList())

        assertThatThrownBy { registry.find(ProviderKind.OPEN_BANKING) }
            .isInstanceOf(IllegalStateException::class.java)
    }
}
