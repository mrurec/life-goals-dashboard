Введение: Почему Jotai?

Recoil был отличной библиотекой, но её разработка замедлилась. Facebook не обновляет его активно, и он не поддерживает React 19 concurrent rendering. Поэтому Life Goals Dashboard мигрировал на Jotai 2.x.

Jotai — это минималистичная библиотека управления состоянием React, созданная для работы с React 19 concurrent rendering. Она использует bottom-up модель атомов (как Recoil), но с чистым API и полной поддержкой Suspense + Server Components.

В Life Goals Dashboard мы используем Jotai для UI-состояния: активная тема, видимость сайдбара, уведомления, параметры виджета. Серверное состояние (проблемы, рецепты, транзакции) управляется Relay, а локальное UI-состояние — Jotai.

Совет: За 1.5 дня вы освоите основные концепции Jotai и научитесь создавать atoms для реальных сценариев.

Introduction: Why Jotai?

Recoil was a great library, but its development slowed down. Meta is not actively updating it, and it does not support React 19 concurrent rendering. That's why Life Goals Dashboard migrated to Jotai 2.x.

Jotai is a minimalist state management library for React, designed for React 19 concurrent rendering. It uses a bottom-up atom model (like Recoil) but with a clean API and full support for Suspense + Server Components.

In Life Goals Dashboard, we use Jotai for UI state: active theme, sidebar visibility, notifications, widget parameters. Server state (problems, recipes, transactions) is managed by Relay, while local UI state is managed by Jotai.

Tip: In 1.5 days, you will master the core concepts of Jotai and learn to create atoms for real-world scenarios.

Core API: Atoms

Atom — это единица состояния в Jotai. Каждый atom имеет значение по умолчанию и необязательный initializer. В отличие от Recoil, атомы в Jotai не требуют уникального key — они идентифицируются объектом.

Три основных хука для работы с атомами:

  • useAtom(atom) — читает И пишет (как useState)
  • useAtomValue(atom) — только читает
  • useSetAtom(atom) — только пишет

Core API: Atoms

Atom is a unit of state in Jotai. Each atom has a default value and an optional initializer. Unlike Recoil, atoms in Jotai do not require a unique key — they are identified by the object itself.

Three main hooks for working with atoms:

  • useAtom(atom) — reads AND writes (like useState)
  • useAtomValue(atom) — read only
  • useSetAtom(atom) — write only

Создание и использование атомовCreating and Using Atoms

import { atom, useAtom, useAtomValue, useSetAtom } from 'jotai';

// 1. Создаём атом с дефолтным значением
const themeAtom = atom<'light' | 'dark'>('dark');
const notificationsAtom = atom<Notification[]>([]);

// 2. useAtom — читает и пишет (как useState)
function ThemeSwitcher() {
  const [theme, setTheme] = useAtom(themeAtom);

  return (
    <button onClick={() => setTheme(theme === 'dark' ? 'light' : 'dark')}>
      Current: {theme}
    </button>
  );
}

// 3. useAtomValue — только читает (оптимизация)
function ThemeDisplay() {
  const theme = useAtomValue(themeAtom);
  return <div className={theme}>Theme: {theme}</div>;
}

// 4. useSetAtom — только пишет (оптимизация)
function NotificationLogger() {
  const setNotifications = useSetAtom(notificationsAtom);

  const logNotification = (msg: string) => {
    setNotifications(prev => [...prev, { id: Date.now(), message: msg }]);
  };

  return <button onClick={() => logNotification('Hello!')}>Log</button>;
}
Совет:Tip: Используйте useAtomValue для компонентов, которые только читают состояние. Это позволит Jotai оптимизировать re-renders и избежать ненужных обновлений.Use useAtomValue for components that only read state. This allows Jotai to optimize re-renders and avoid unnecessary updates.

Производные атомы (Derived Atoms)

Производный атом — это атом, значение которого зависит от других атомов. Есть два вида:

  1. Только для чтения (read-only): вычисляет значение на основе других атомов
  2. С возможностью записи (writable): может быть обновлён и также обновить зависимые атомы

Derived Atoms

Derived atom is an atom whose value depends on other atoms. There are two types:

  1. Read-only: computes value based on other atoms
  2. Writable: can be updated and also update dependent atoms

Пример: Read-only atomExample: Read-only atom

import { atom } from 'jotai';

const notificationsAtom = atom<Notification[]>([]);

// Производный атом — только для чтения
// Вычисляет количество непрочитанных уведомлений
const unreadCountAtom = atom((get) => {
  const notifications = get(notificationsAtom);
  return notifications.filter(n => !n.read).length;
});

function NotificationBell() {
  const unreadCount = useAtomValue(unreadCountAtom);
  return (
    <button>
      🔔 {unreadCount > 0 && <span className="badge">{unreadCount}</span>}
    </button>
  );
}

Пример: Writable atomExample: Writable atom

// Производный атом с возможностью записи
// Может как читать, так и обновлять другие атомы
const userDisplayNameAtom = atom(
  (get) => {
    const user = get(currentUserAtom);
    return user?.name || 'Anonymous';
  },
  (get, set, newName: string) => {
    const user = get(currentUserAtom);
    if (user) {
      set(currentUserAtom, { ...user, name: newName });
    }
  }
);

function UserGreeting() {
  const [displayName, setDisplayName] = useAtom(userDisplayNameAtom);

  return (
    <>
      <p>Welcome, {displayName}!</p>
      <input
        value={displayName}
        onChange={(e) => setDisplayName(e.target.value)}
      />
    </>
  );
}

atomFamily: Параметризованные атомы

atomFamily позволяет создавать семейства атомов, параметризованные по ID или ключу. Это идеально для списков, где каждый элемент имеет свой独立 state (например, открытость каждого widget на dashboard).

atomFamily: Parameterized Atoms

atomFamily allows you to create families of atoms, parameterized by ID or key. This is perfect for lists where each item has independent state (e.g., the open/closed state of each widget on the dashboard).

import { atomFamily } from 'jotai/utils';

// Семейство атомов для видимости каждого widget
// Каждый widget имеет свой atom на основе его ID
const widgetVisibilityAtom = atomFamily((widgetId: string) =>
  atom(true) // дефолтно видим
);

function Dashboard() {
  const widgets = [
    { id: 'fitness', title: 'Fitness Tracker' },
    { id: 'budget', title: 'Budget Planner' },
    { id: 'interview', title: 'Interview Prep' },
  ];

  return (
    <div>
      {widgets.map(widget => (
        <WidgetCard
          key={widget.id}
          id={widget.id}
          title={widget.title}
        />
      ))}
    </div>
  );
}

function WidgetCard({ id, title }: { id: string; title: string }) {
  const [isVisible, setIsVisible] = useAtom(widgetVisibilityAtom(id));

  if (!isVisible) return null;

  return (
    <div className="widget">
      <h3>{title}</h3>
      <button onClick={() => setIsVisible(false)}>Hide</button>
    </div>
  );
}

atomWithStorage: Persistence

atomWithStorage автоматически сохраняет значение atom в localStorage (или sessionStorage). Идеально для пользовательских предпочтений, которые должны сохраняться между сессиями.

atomWithStorage: Persistence

atomWithStorage automatically saves the atom value to localStorage (or sessionStorage). Perfect for user preferences that should persist between sessions.

import { atomWithStorage } from 'jotai/utils';

// Атом с автоматической persistence в localStorage
const themeAtom = atomWithStorage<'light' | 'dark'>('lifegoals:theme', 'dark');
const sidebarCollapsedAtom = atomWithStorage('lifegoals:sidebarCollapsed', false);

function ThemeSwitcher() {
  const [theme, setTheme] = useAtom(themeAtom);

  // При первом рендере загружается из localStorage
  // При изменении автоматически сохраняется
  return (
    <button onClick={() => setTheme(theme === 'dark' ? 'light' : 'dark')}>
      Toggle {theme}
    </button>
  );
}

function Layout() {
  const [collapsed, setCollapsed] = useAtom(sidebarCollapsedAtom);

  return (
    <div className={collapsed ? 'layout-collapsed' : 'layout-expanded'}>
      {/* Sidebar, content... */}
    </div>
  );
}

Асинхронные атомы & Suspense

Jotai отлично интегрируется с Suspense. Атомы, возвращающие Promise, автоматически работают с <Suspense>.

Async Atoms & Suspense

Jotai integrates perfectly with Suspense. Atoms that return a Promise automatically work with <Suspense>.

import { atom } from 'jotai';
import { Suspense } from 'react';

// Асинхронный атом — возвращает Promise
const currentUserAtom = atom(async () => {
  const res = await fetch('/api/me');
  return res.json();
});

function UserProfile() {
  const user = useAtomValue(currentUserAtom);

  // useAtomValue автоматически ждет Promise
  // Suspense показывает fallback пока загружается
  return <div>User: {user.name}</div>;
}

// Компонент с Suspense
export function UserProfilePage() {
  return (
    <Suspense fallback={<div>Loading user...</div>}>
      <UserProfile />
    </Suspense>
  );
}

Provider & Тестирование

Jotai использует <Provider> для подключения атомов к приложению. Для тестирования можно создавать отдельные Provider экземпляры с инициальным состоянием.

Provider & Testing

Jotai uses <Provider> to connect atoms to your app. For testing, you can create separate Provider instances with initial state.

import { Provider } from 'jotai';
import { render, screen } from '@testing-library/react';

// В App.tsx
export function App() {
  return (
    <Provider>
      <Dashboard />
    </Provider>
  );
}

// В тесте
describe('Theme Switcher', () => {
  it('should toggle theme', () => {
    const { rerender } = render(
      <Provider>
        <ThemeSwitcher />
      </Provider>
    );

    const button = screen.getByRole('button');
    fireEvent.click(button);

    expect(button).toHaveTextContent('light');
  });

  it('should initialize with stored value', () => {
    localStorage.setItem('lifegoals:theme', 'light');

    render(
      <Provider>
        <ThemeSwitcher />
      </Provider>
    );

    expect(screen.getByRole('button')).toHaveTextContent('light');
  });
});

describe('Theme Atom', () => {

import { Provider } from 'jotai';
import { useAtom, useAtomValue } from 'jotai';

const notificationsAtom = atom<Array<{ id: string; message: string; read: boolean }>>([]);

Описание:Description: Создайте асинхронный atom, который загружает текущего пользователя с сервера. Используйте Suspense для обработки загрузки.Create an async atom that loads the current user from the server. Use Suspense to handle loading.

import { render, screen } from '@testing-library/react';
import { Provider } from 'jotai';
import { useAtomValue } from 'jotai';
import { atom } from 'jotai';
import { Suspense } from 'react';

const currentUserAtom = atom(async () => {
  const res = await fetch('/api/me');
  if (!res.ok) throw new Error('Failed to load user');
  return res.json();
});

describe('Current User Async Atom', () => {
  beforeEach(() => {
    global.fetch = jest.fn();
  });

  it('should load user data', async () => {
    (global.fetch as jest.Mock).mockResolvedValueOnce({
      ok: true,
      json: async () => ({ id: '1', name: 'Test User' }),
    });

    const UserProfile = () => {
      const user = useAtomValue(currentUserAtom);
      return <div>User: {user.name}</div>;
    };

    render(
      <Provider>
        <Suspense fallback={<div>Loading...</div>}>
          <UserProfile />
        </Suspense>
      </Provider>
    );

    expect(screen.getByText('Loading...')).toBeInTheDocument();
    await screen.findByText('User: Test User');
  });

  it('should handle fetch errors', async () => {
    (global.fetch as jest.Mock).mockResolvedValueOnce({
      ok: false,
    });

    const UserProfile = () => {
      try {
        const user = useAtomValue(currentUserAtom);
        return <div>User: {user.name}</div>;
      } catch (e) {
        return <div>Error: {(e as Error).message}</div>;
      }
    };

    render(
      <Provider>
        <Suspense fallback={<div>Loading...</div>}>
          <UserProfile />
        </Suspense>
      </Provider>
    );

    await screen.findByText(/Error/);
  });
});
