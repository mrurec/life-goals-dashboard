Введение: Redis для Life Goals DashboardIntroduction: Redis for Life Goals Dashboard

Redis — это высокопроизводительное хранилище данных в памяти (in-memory data store), которое используется для кеширования и pub/sub систем. В Life Goals Dashboard Redis кеширует часто используемые данные (курсы обмена валют, статистику проблем) и доставляет notifications в реальном времени через WebSocket.

За 2 дня вы освоите Redis data structures, поймёте стратегии кеширования (Cache-Aside, Write-Through), реализуете pub/sub для GraphQL Subscriptions и интегрируете Redis в Spring Boot приложение.

Совет: Установите Redis локально (`redis-server`) или используйте Docker: docker run -d -p 6379:6379 redis:7. Тестируйте в Redis CLI (`redis-cli`).

Redis is a high-performance in-memory data store used for caching and pub/sub systems. In Life Goals Dashboard, Redis caches frequently accessed data (exchange rates, problem statistics) and delivers real-time notifications via WebSocket.

Over 2 days you will master Redis data structures, understand caching strategies (Cache-Aside, Write-Through), implement pub/sub for GraphQL Subscriptions, and integrate Redis into a Spring Boot application.

Tip: Install Redis locally (`redis-server`) or use Docker: docker run -d -p 6379:6379 redis:7. Test in Redis CLI (`redis-cli`).

Правила именования ключейKey Naming Rules

В Life Goals Dashboard следуем строгому паттерну именования ключей:

In Life Goals Dashboard we follow a strict key naming pattern: 


lifegoals:{widget_type}:{entity}:{id_or_query_hash}

Компоненты:

  • lifegoals — префикс приложения (избегает коллизии в Redis если используется несколько приложений)
  • widget_type — тип виджета: `budget`, `fitness`, `interviewprep`
  • entity — тип сущности: `exchange_rate`, `problem_stats`, `food_product`, `dashboard_widget`
  • id_or_query_hash — идентификатор (UUID, пара валют, хеш запроса)

Примеры реальных ключей:

Components:

  • lifegoals — application prefix (avoids collisions in Redis if multiple apps use it)
  • widget_type — widget type: `budget`, `fitness`, `interviewprep`
  • entity — entity type: `exchange_rate`, `problem_stats`, `food_product`, `dashboard_widget`
  • id_or_query_hash — identifier (UUID, currency pair, query hash)

Real key examples:

Стратегии кешированияCaching Strategies

Cache-Aside (Lazy Loading)Cache-Aside (Lazy Loading)

Клиент проверяет кеш. Если данных нет — получает из БД, сохраняет в кеш и возвращает. Кеш автоматически истекает по TTL.

Блок-схема:

Client checks cache. If data doesn't exist, fetch from database, save to cache, and return. Cache expires automatically by TTL.

Flowchart:

  1. Запрос данных (например, курс EUR → KRW)
  2. Проверить Redis GET ключ → если есть, вернуть
  3. Если нет → получить из ExchangeRate API
  4. SETEX в Redis с TTL 1 час
  5. Вернуть клиенту

Пример: ExchangeRateCacheService в Kotlin

  1. Request data (e.g., EUR → KRW rate)
  2. Check Redis GET key → if exists, return
  3. If not → fetch from ExchangeRate API
  4. SETEX to Redis with 1-hour TTL
  5. Return to client

Example: ExchangeRateCacheService in Kotlin


@Service
class ExchangeRateCacheService(
    private val redisTemplate: RedisTemplate<String, String>,
    private val exchangeRateApi: ExchangeRateApi
) {
    private val TTL_SECONDS = 3600L  // 1 час

    fun getExchangeRate(baseCurrency: String, targetCurrency: String): BigDecimal {
        val cacheKey = "lifegoals:budget:exchange_rate:$baseCurrency:$targetCurrency"

        // 1. Проверить Redis
        val cachedRate = redisTemplate.opsForValue().get(cacheKey)
        if (cachedRate != null) {
            return BigDecimal(cachedRate)
        }

        // 2. Получить из API
        val rate = exchangeRateApi.fetchRate(baseCurrency, targetCurrency)

        // 3. Сохранить в кеш на 1 час
        redisTemplate.opsForValue().set(
            cacheKey,
            rate.toPlainString(),
            Duration.ofSeconds(TTL_SECONDS)
        )

        return rate
    }
}

Write-Through (Запись сквозь кеш)Write-Through

При обновлении данных: сначала пишется в БД, затем в кеш. Кеш всегда валидный. При мутации кеш инвалидируется (DEL) и пересчитывается при следующем запросе.

Пример: Dashboard Widget Cache Invalidation

On data update: write to DB first, then to cache. Cache is always valid. On mutation, cache is invalidated (DEL) and recalculated on next request.

Example: Dashboard Widget Cache Invalidation


@Service
class DashboardWidgetService(
    private val redisTemplate: RedisTemplate<String, String>,
    private val widgetRepository: DashboardWidgetRepository
) {
    fun updateWidgetConfig(widgetId: UUID, newConfig: JsonNode): DashboardWidget {
        // 1. Обновить в БД
        val updated = widgetRepository.save(
            DashboardWidget(
                id = widgetId,
                config = newConfig,
                updatedAt = Instant.now()
            )
        )

        // 2. Инвалидировать кеш (DELETE из Redis)
        val cacheKey = "lifegoals:dashboard:widget_data:$widgetId"
        redisTemplate.delete(cacheKey)

        // Следующий запрос пересчитает данные с новым конфигом
        return updated
    }
}

Redis Pub/Sub для GraphQL SubscriptionsRedis Pub/Sub for GraphQL Subscriptions

Архитектура Pub/SubPub/Sub Architecture

Redis Pub/Sub — это система "издатель-подписчик". Когда происходит событие (например, логирование тренировки), сервис публикует сообщение в Redis канал. WebSocket подписчики получают сообщение в реальном времени.

Поток данных:

Redis Pub/Sub is a "publisher-subscriber" system. When an event occurs (e.g., logging a workout), the service publishes a message to a Redis channel. WebSocket subscribers receive the message in real-time.

Data flow:


Client (WebSocket) ← GraphQL DGS Subscription ← Redis SUBSCRIBE ← Service (PUBLISH)

Redis SUBSCRIBE иand PUBLISH


# Подписаться на канал (слушатель)
SUBSCRIBE channel-name
SUBSCRIBE lifegoals:fitness:notifications

# Опубликовать сообщение (издатель)
PUBLISH channel-name message
PUBLISH lifegoals:fitness:notifications '{"type": "WORKOUT_LOGGED", "userId": "123", "workoutId": "456"}'

# Подписаться на паттерн каналов
PSUBSCRIBE lifegoals:*:notifications

        )

        return saved
    }
}

@Configuration
class RedisConfig {
    @Bean
    fun redisTemplate(connectionFactory: LettuceConnectionFactory): RedisTemplate<String, String> {
        val template = RedisTemplate<String, String>()
        template.setConnectionFactory(connectionFactory)

        // Serializers для ключей и значений
        val stringSerializer = StringRedisSerializer()
        val jsonSerializer = Jackson2JsonRedisSerializer(String::class.java)

        template.keySerializer = stringSerializer
        template.valueSerializer = jsonSerializer
        template.hashKeySerializer = stringSerializer
        template.hashValueSerializer = jsonSerializer

        return template
    }
}

Использование в сервисеUsage in Service


@Service
class CacheService(
    private val redisTemplate: RedisTemplate<String, String>
) {
    fun cache(key: String, value: String, ttlSeconds: Long) {
        redisTemplate.opsForValue().set(
            key,
            value,
            Duration.ofSeconds(ttlSeconds)
        )
    }

    fun get(key: String): String? {
        return redisTemplate.opsForValue().get(key)
    }

    fun delete(key: String) {
        redisTemplate.delete(key)
    }

    fun increment(key: String, value: Long = 1): Long {
        return redisTemplate.opsForValue().increment(key, value) ?: 0
    }
}

package com.mrurec.lifegoals.common.cache

import io.mockk.*
import org.junit.jupiter.api.BeforeEach
import org.junit.jupiter.api.Test
import org.junit.jupiter.api.Assertions.*
import org.springframework.data.redis.core.RedisTemplate
import org.springframework.data.redis.core.ValueOperations
import java.time.Duration

class CacheServiceTest {

    private val redisTemplate = mockk<RedisTemplate<String, Any>>()
    private val valueOps = mockk<ValueOperations<String, Any>>()
    private lateinit var cacheService: CacheService

    @BeforeEach
    fun setup() {
        every { redisTemplate.opsForValue() } returns valueOps
        cacheService = CacheService(redisTemplate)
    }

    @Test
    fun `get should return cached value when exists`() {
        every { valueOps.get("lifegoals:budget:exchange_rate:EUR:KRW") } returns 1450.5

        val result = cacheService.get<Double>("lifegoals:budget:exchange_rate:EUR:KRW")

        assertEquals(1450.5, result)
    }

    @Test
    fun `get should return null when key not found`() {
        every { valueOps.get("lifegoals:fitness:workout:nonexistent") } returns null

        val result = cacheService.get<Any>("lifegoals:fitness:workout:nonexistent")

        assertNull(result)
    }

    @Test
    fun `set should store value with TTL`() {
        every { valueOps.set(any(), any(), any<Duration>()) } just Runs

        cacheService.set("lifegoals:budget:exchange_rate:EUR:KRW", 1450.5, Duration.ofHours(1))

        verify { valueOps.set("lifegoals:budget:exchange_rate:EUR:KRW", 1450.5, Duration.ofHours(1)) }
    }

    @Test
    fun `evict should delete specific key`() {
        every { redisTemplate.delete(any<String>()) } returns true

        cacheService.evict("lifegoals:fitness:workout:123")

        verify { redisTemplate.delete("lifegoals:fitness:workout:123") }
    }

    @Test
    fun `evictByPrefix should delete all keys matching prefix`() {
        val keys = setOf("lifegoals:interviewprep:problem:1", "lifegoals:interviewprep:problem:2")
        every { redisTemplate.keys("lifegoals:interviewprep:problem:*") } returns keys
        every { redisTemplate.delete(keys) } returns 2

        cacheService.evictByPrefix("lifegoals:interviewprep:problem")

        verify { redisTemplate.keys("lifegoals:interviewprep:problem:*") }
        verify { redisTemplate.delete(keys) }
    }

    @Test
    fun `key pattern should follow convention lifegoals_module_entity_id`() {
        val key = CacheService.buildKey("budget", "exchange_rate", "EUR", "KRW")
        assertEquals("lifegoals:budget:exchange_rate:EUR:KRW", key)
    }
}

import org.junit.jupiter.api.Test
import org.junit.jupiter.api.Assertions.*
import org.springframework.beans.factory.annotation.Autowired
import org.springframework.boot.test.context.SpringBootTest
import org.springframework.data.redis.core.RedisTemplate
import org.springframework.data.redis.serializer.GenericJackson2JsonRedisSerializer
import org.springframework.data.redis.serializer.StringRedisSerializer
import org.springframework.test.context.DynamicPropertyRegistry
import org.springframework.test.context.DynamicPropertySource
import org.testcontainers.containers.GenericContainer
import org.testcontainers.junit.jupiter.Container
import org.testcontainers.junit.jupiter.Testcontainers

@SpringBootTest
@Testcontainers
class RedisConfigTest {

    companion object {
        @Container
        val redis = GenericContainer("redis:7-alpine").withExposedPorts(6379)

        @DynamicPropertySource
        @JvmStatic
        fun configureProperties(registry: DynamicPropertyRegistry) {
            registry.add("spring.data.redis.host") { redis.host }
            registry.add("spring.data.redis.port") { redis.getMappedPort(6379) }
        }
    }

    @Autowired
    lateinit var redisTemplate: RedisTemplate<String, Any>

    @Test
    fun `RedisTemplate should use String key serializer`() {
        assertTrue(redisTemplate.keySerializer is StringRedisSerializer)
    }

    @Test
    fun `RedisTemplate should use Jackson JSON value serializer`() {
        assertTrue(redisTemplate.valueSerializer is GenericJackson2JsonRedisSerializer)
    }

    @Test
    fun `should be able to set and get a value`() {
        redisTemplate.opsForValue().set("test:key", "hello")
        val result = redisTemplate.opsForValue().get("test:key")
        assertEquals("hello", result)
    }
}
