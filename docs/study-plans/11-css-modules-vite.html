Introduction: CSS Modules + Vite

CSS Modules is a way to write CSS with local scope. Instead of global class names (which conflict), each `.module.css` file has a unique namespace.

Vite 8.0.8 is a modern build tool that replaces Webpack. Very fast, modern syntax, built-in CSS Modules support. New: Rolldown (Rust-based bundler) — 10-30× faster than the old esbuild+Rollup split, unified dev & build.

In 1.5 days you will master:

  • CSS Modules: .module.css files, import as object, scoped classes
  • Vite configuration: React plugin, Relay plugin, GraphQL proxy
  • CSS variables (Custom Properties) for theming
  • Optimization: code splitting, image optimization
Tip: CSS Modules solve the global scope problem in CSS. No need to invent unique names like "ComponentNameButtonPrimary". Just use ".button" and CSS Modules will make it unique.

import react from '@vitejs/plugin-react';

export default defineConfig({
    plugins: [react()],
    server: {
        port: 5173,
        proxy: {
            // Проксируем GraphQL запросы на backend
            // Proxy GraphQL requests to backend
            '/graphql': {
                target: 'http://localhost:8080',
                changeOrigin: true,
                secure: false
            },
            // Проксируем WebSocket subscriptions
            // Proxy WebSocket subscriptions
            '/subscriptions': {
                target: 'ws://localhost:8080',
                ws: true
            }
        }
    },
    build: {
        outDir: 'dist',
        sourcemap: false,  // Отключите sourcemap в prod для меньшего размера
        rollupOptions: {
            output: {
                manualChunks: {
                    // Code splitting: отдельный чанк для Relay
                    'relay': ['relay-runtime']
                }
            }
        }
    },
    css: {
        modules: {
            localsConvention: 'camelCase'  // CSS классы в camelCase
        }
    }
});

Best Practices

File Structure


src/

└── main.tsx

Правила

  1. Один .module.css на компонент — коллокализация (рядом с .tsx)
  2. Используйте CSS переменные для всех цветов, размеров, spacing
  3. Избегайте глобальных классов — всё должно быть в modules
  4. camelCase для классов в CSS файлах, которые станут camelCase в import'е
  5. Не дублируйте стили — используйте `composes` для наследования

Оптимизация: Code Splitting

Vite автоматически сплитует код, но можете оптимизировать вручную:

Rules

  1. One .module.css per component — colocation (next to .tsx)
  2. Use CSS variables for all colors, sizes, spacing
  3. Avoid global classes — everything should be in modules
  4. camelCase for classes in CSS files that become camelCase in imports
  5. Don't duplicate styles — use `composes` for inheritance

Optimization: Code Splitting
